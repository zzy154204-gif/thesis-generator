\subsection{缺陷根因分析}

采用5Why分析法对代表性缺陷进行根因追溯：

\textbf{BUG-01（PDF导出依赖LibreOffice）}：
\begin{itemize}
    \item Why 1：为什么PDF导出依赖LibreOffice？→ 无Java原生PDF渲染引擎
    \item Why 2：为什么不使用Java原生方案？→ 开发周期内优先保证功能可用性
    \item Why 3：后续如何优化？→ 改用Apache PDFBox + FlyingSaucer替代，可减少镜像200MB
\end{itemize}

\textbf{BUG-02至05} 均为构建工具链配置问题，根本原因是项目初期未建立统一的开发环境配置模板，各成员本地环境差异导致部分依赖缺失。已在 \texttt{dev-memberD-v2} 分支中通过补全 \texttt{settings.xml}（Maven镜像）、更新 \texttt{vite.config.ts}（Sass配置）、安装缺失npm包等方式全部修复。

\subsection{未修复遗留缺陷说明}

BUG-01（LibreOffice依赖）为架构级设计取舍，在实训项目周期内不阻断功能使用，建议在后续版本中优化。当前Docker镜像已包含LibreOffice，功能可正常使用。无其他遗留缺陷。

\subsection{功能完整性结论}

系统完整覆盖需求规格说明书中定义的全部功能需求，9大功能模块和12个前端页面全部通过测试。核心业务流程（注册→登录→创建论文→编辑章节→添加参考文献→预览→DOCX/PDF导出→教师审核）可完整走通。功能覆盖率100\%。

\subsection{性能达标情况结论}

本地Docker部署环境下，核心接口在50并发以内响应时间<150ms，QPS>300。Docker三容器总内存占用约720MB。PDF导出单次耗时1-5秒（取决于内容复杂度）。各项指标满足高校实训项目的实用性要求，系统可在2GB以上内存的常规机器上流畅运行。

\subsection{系统安全与兼容性结论}

\begin{itemize}
    \item \textbf{安全}：16项安全测试全部通过，JWT+RABC权限模型正确，SQL/XSS注入防护有效，BCrypt密码加密到位，无P0/P1级安全缺陷
    \item \textbf{兼容}：Chrome/Edge/Firefox/Safari四大浏览器均通过，PostgreSQL/H2双数据库兼容，Docker和本地双部署模式均验证通过
\end{itemize}

\subsection{上线/交付判定意见}

\textbf{判定结论：系统达到交付标准，建议上线。}

全部122条测试用例通过率96.7\%（超过预设90\%阈值），P0/P1级缺陷数为0，P2级缺陷1个已有规避方案（镜像内置LibreOffice），功能需求覆盖率100\%。建议在以下条件满足后上线：
\begin{enumerate}
    \item JWT Secret改用环境变量注入（当前硬编码在配置文件中）
    \item 生产环境启用HTTPS
    \item 补充至少2个核心流程的自动化回归测试脚本
\end{enumerate}

\section{建议与改进措施}

\subsection{程序代码优化建议}
\begin{enumerate}
    \item PDF导出模块考虑用Apache PDFBox + FlyingSaucer替代LibreOffice，减小Docker镜像体积约200MB
    \item 为papers表title字段增加数据库索引，优化搜索性能（当前万级数据查询耗时2.8s）
    \item 编辑器图片上传增加进度条反馈，提升大文件上传体验
    \item 建议统一使用\texttt{dev-memberD-v2}分支的构建配置（已含Maven镜像和完整依赖）
\end{enumerate}

\subsection{业务逻辑优化建议}
\begin{enumerate}
    \item 论文列表增加学院、状态筛选项，提升用户检索效率
    \item 登录失败增加锁定机制（5次失败锁定30分钟），当前未实现
    \item API增加Rate Limiting中间件，防止暴力破解和滥用
\end{enumerate}

\subsection{后续迭代测试优化建议}
\begin{enumerate}
    \item 引入JUnit 5 + Spring Boot Test编写后端单元测试和集成测试（当前测试覆盖率为0\%）
    \item 引入Selenium + Vitest编写前端自动化E2E测试，覆盖核心业务流程
    \item 添加Spring Boot Actuator + Prometheus监控指标暴露，便于生产环境运维
    \item CI/CD流水线中集成自动构建+测试环节，确保每次提交通过编译和基础测试
\end{enumerate}
