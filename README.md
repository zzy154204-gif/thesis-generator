# 小组项目协作说明

## 环境要求
- Windows 10/11
- MiKTeX（已通过 winget 安装）
- Git

## 快速开始

### 1. 编译文档
双击 `build.ps1` 或在终端运行：
```powershell
.\build.ps1
```

### 2. Git 工作流（小组成员必读）

**首次克隆**
```bash
git clone <远程仓库地址>
```

**每次写文档前**
```bash
git pull   # 拉取队友最新修改
```

**写完后提交**
```bash
git add .
git commit -m "描述你改了什么，例如：补充功能需求模块二"
git push
```

**冲突了怎么办**
- 不要慌，Git 会标出冲突位置
- 和队友沟通保留谁的版本
- 解决后重新 add → commit → push

### 3. 分支建议
```bash
git checkout -b feature/你的名字-功能名   # 创建个人分支
# 写完合并
git checkout master
git merge feature/你的名字-功能名
```

## 文件说明

| 文件 | 用途 |
|------|------|
| `需求分析文档.tex` | LaTeX 源码，编辑这个 |
| `需求分析文档.pdf` | 编译输出，最终交付物 |
| `build.ps1` | 一键编译脚本 |
| `.gitignore` | 忽略编译中间文件 |

## 需求分析文档结构

1. **引言** — 为什么要做这个项目、给谁看
2. **系统概述** — 系统目标、用户角色、运行环境
3. **功能需求** — 每个功能模块的详细规格（编号、优先级、输入输出、前置后置条件）
4. **非功能需求** — 性能、安全性、可用性、可维护性
5. **用例描述** — 用例图 + 用例规约表
6. **数据需求** — 数据实体、ER图
7. **验收标准** — 项目完成的判定条件
