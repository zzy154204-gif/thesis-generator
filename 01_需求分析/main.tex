\section{引言}
\subsection{编写目的}
本文档为在线论文文档自动生成系统软件需求规格说明书，用于完整定义系统全部功能性与非功能性需求，明确项目开发边界、业务规则、数据规范、运行约束。
本文档交付对象包含项目经理、系统设计人员、前后端开发工程师、测试人员，作为项目规划、架构设计、代码开发、功能测试、验收交付的核心依据。

\subsection{项目背景}
本系统为高校大二软件工程实训项目，开发单位为在校实训开发小组，服务对象为高校本科及研究生毕业生与论文指导教师。
当前高校毕业论文排版流程繁琐，学生对Word格式调整、目录生成、参考文献管理、公式编号等操作不熟悉，导致大量时间耗费在格式调整而非内容撰写上。本系统独立部署运行，依托Web服务与文档自动生成引擎，帮助学生在网页端填入论文内容，系统自动生成符合学校排版规范的完整论文文档（PDF/Word），大幅降低论文排版门槛，提升毕业论文撰写效率。

\subsection{定义}
\begin{itemize}
    \item SRS：Software Requirements Specification，软件需求规格说明书；
    \item LaTeX：一种专业的文档排版系统，广泛用于学术论文撰写；
    \item 论文模板：符合学校教务要求的预设文档格式模板，包含封面、目录、页眉页脚、字体字号、行间距等规范；
    \item 自动排版引擎：根据用户输入的结构化内容，自动套用模板生成格式化文档的核心模块；
    \item 参考文献管理：支持GB/T 7714等标准格式的文献引用与自动排序功能；
    \item 目录生成：根据章节标题自动提取并生成带页码的目录。
\end{itemize}

\subsection{参考资料}
\begin{enumerate}
    \item GB/T 9385-2008 计算机软件需求规格说明规范；
    \item 实训项目任务书（2026软件工程实训）；
    \item Web前后端分离开发规范文档；
    \item 高校本科毕业论文（设计）撰写规范；
    \item GB/T 7714-2015 信息与文献 参考文献著录规则。
\end{enumerate}

\section{任务概述}
\subsection{目标}
搭建一套Web端在线论文文档自动生成平台，区分学生、指导教师、系统管理员三类角色；实现论文模板管理、章节内容在线编辑、自动排版生成（封面/目录/正文/参考文献）、文档导出（PDF/Word）、参考文献自动编号排序、公式与图表自动编号、指导教师在线审阅批注功能；管理员端提供模板管理、用户管理、系统参数配置功能，满足高校毕业论文在线撰写与规范化排版的业务需求。

\subsection{运行环境}
\begin{itemize}
    \item 服务操作系统：Windows Server / Linux CentOS；
    \item Web服务：Nginx + Spring Boot；
    \item 数据库：MySQL 8.0；
    \item 文档生成引擎：基于Pandoc + LaTeX模板的自动排版；
    \item 文件存储：本地存储 + 云存储（OSS）备份；
    \item 前端框架：Vue 3 + Element Plus；
    \item 客户端：主流Chrome、Edge浏览器。
\end{itemize}

\subsection{条件与限制}
\begin{enumerate}
    \item 系统需支持至少50名学生同时在线编辑论文；
    \item 单篇论文文档生成（PDF/Word）时间不超过10秒；
    \item 论文模板须严格符合学校教务处的排版规范，不得出现格式偏差；
    \item 仅支持校内师生凭学号/工号登录，不开放校外用户；
    \item 系统需与学校统一认证系统对接（学号/工号单点登录）；
    \item 论文内容数据须加密存储，确保学术信息安全。
\end{enumerate}

\section{数据描述}
\subsection{静态数据}
系统长期持久存储不变基础数据：用户账号信息（学生/教师/管理员）、论文模板配置（封面样式、字体字号、页边距、行间距、页眉页脚格式）、学校学院信息、学位类型分类（本科/硕士）。

\subsection{动态数据}
系统运行过程中实时产生、变更的数据：用户登录会话、论文草稿内容（各章节文本、图片、表格、公式）、文档生成任务队列、生成历史记录、教师审阅批注、论文版本快照、导出文件记录、操作日志。

\subsection{数据库介绍}
采用MySQL 8.0关系型数据库存储全部业务数据，划分用户表（user）、论文表（thesis）、章节表（chapter）、模板表（template）、批注表（comment）、生成记录表（generation\_log）、系统日志表（sys\_log），数据表通过外键建立关联约束。

\subsection{数据词典}
包含用户ID（学号/工号）、姓名、角色类型、论文ID、论文标题、学位类型、章节标题、章节内容（富文本）、模板ID、模板名称、图片路径、参考文献条目、批注内容、生成时间、导出格式、文件存储路径等全部字段含义、数据类型、取值范围说明。

\subsection{数据采集}
\begin{itemize}
    \item 用户数据：对接学校统一认证系统，自动同步师生信息；
    \item 论文内容：学生通过网页端富文本编辑器在线撰写并实时保存；
    \item 模板数据：管理员通过后台配置并上传模板文件；
    \item 生成记录：每次文档生成操作自动记录生成时间、格式、结果。
\end{itemize}

\section{功能需求}
\subsection{功能划分}
系统整体划分为五大功能模块：论文撰写与编辑模块、自动排版生成模块、教师审阅模块、模板管理模块、系统管理模块。

\subsection{功能描述}

% --- 论文撰写与编辑 ---
\reqitem{FR001}{学生可通过学号/工号登录系统，创建新论文项目，填写论文标题、选择学位类型与模板。}
\reqitem{FR002}{学生可在线编辑论文各章节（摘要、引言、正文、结论等），支持富文本编辑（加粗、斜体、列表、插入图片、插入表格、数学公式）。}
\reqitem{FR003}{学生可添加、编辑、删除参考文献条目，系统自动按GB/T 7714标准格式化并排序。}
\reqitem{FR004}{系统自动保存论文草稿，支持查看历史版本并回滚至任意历史版本。}

% --- 自动排版生成 ---
\reqitem{FR005}{系统根据所选模板，自动生成包含封面、目录、正文、参考文献的完整论文文档。}
\reqitem{FR006}{系统自动为论文中的图表、公式生成编号（如图1-1、表2-3），并更新目录中的图表索引。}
\reqitem{FR007}{系统支持将论文导出为PDF格式（符合学校打印规范）和Word格式（方便修改）。}
\reqitem{FR008}{系统在生成文档前自动进行格式校验，检测缺失章节、空参考文献、未替换占位符等常见问题并提示。}

% --- 教师审阅 ---
\reqitem{FR009}{指导教师可查看所指导学生的论文内容，在线添加批注和修改建议。}
\reqitem{FR010}{教师可在论文指定位置添加高亮标记和文字批注，学生端实时可见。}

% --- 模板管理 ---
\reqitem{FR011}{管理员可上传、编辑、删除论文模板，包括封面样式、字体配置、页眉页脚样式等。}
\reqitem{FR012}{系统支持多种学位类型模板（本科、硕士），学生创建论文时选择对应模板。}

% --- 系统管理 ---
\reqitem{FR013}{系统管理员可管理用户账号、分配指导教师与学生对应关系。}
\reqitem{FR014}{系统管理员可查看操作日志，追溯所有论文生成、编辑与导出记录。}

\section{性能需求}
\subsection{数据精确度}
论文排版输出必须严格匹配模板规范，字体、字号、行间距、页边距、页码格式零偏差；参考文献排序严格按照GB/T 7714标准，无错排、漏排；公式编号与目录页码自动更新，无错位。

\subsection{时间特性}
\begin{itemize}
    \item 网页页面加载响应时间 $\le$ 1.5s；
    \item 章节内容自动保存间隔 30s；
    \item 单篇论文文档生成时间（含封面/目录/正文/参考文献）$\le$ 10s；
    \item 历史版本回滚操作响应时间 $\le$ 2s。
\end{itemize}

\subsection{适应性}
系统可适配Windows、Linux服务端环境；支持新增论文模板（如不同学院、不同学位类型）；支持扩展文档导出格式（如HTML、LaTeX源码）；可通过更新模板配置适配学校每年更新的排版规范。

\section{运行需求}
\subsection{用户界面}
采用Web网页可视化界面，响应式布局适配PC端。学生端：论文项目列表（论文标题+进度+最后编辑时间）、章节编辑页面（左侧章节目录树 + 右侧富文本编辑器 + 实时字数统计）、参考文献管理面板、生成预览窗口。教师端：指导学���列表、论文审阅页面（论文内容展示 + 批注侧栏）。管理端：模板管理表格、用户管理列表、系统配置面板。

\subsection{硬件接口}
无需专用硬件外设接口，仅依赖服务器CPU、内存、磁盘资源完成文档编译与数据存储。文档生成引擎需要服务器具备LaTeX/ Pandoc运行环境。

\subsection{软件接口}
\begin{itemize}
    \item 数据库接口：MySQL标准JDBC连接接口；
    \item 前端接口：前后端RESTful API数据交互接口，JSON格式；
    \item 学校统一认证接口：CAS/OAuth 2.0协议，实现学号/工号单点登录；
    \item 文档生成接口：本地进程调用Pandoc + XeLaTeX编译链；
    \item 文件存储接口：本地文件系统 + OSS云存储SDK。
\end{itemize}

\subsection{故障处理}
\begin{itemize}
    \item 数据库断开：前端页面提示"服务暂不可用"，未保存内容缓存至浏览器本地存储；
    \item 文档生成失败：前端提示具体错误原因（如缺少必需章节、模板损坏），任务可重试；
    \item 文件导出超时：大论文生成采用异步任务模式，完成后通过站内消息通知下载；
    \item 网络中断：编辑器自动保存草稿，恢复连接后同步至服务器。
\end{itemize}

\section{其它需求}
\begin{enumerate}
    \item 安全性：区分学生、教师、管理员三级角色权限；密码采用BCrypt加密存储；论文内容数据加密存储，防止学术泄露；SQL参数化查询防止注入攻击；敏感操作记录操作日志；
    \item 可维护性：后台提供可视化管理页面，代码遵循MVC分层架构，模板修改不需要重新部署系统；
    \item 可移植性：基于Java Spring Boot + Vue前后端分离架构，可快速部署至不同服务器环境；
    \item 易用性：学生无需学习LaTeX或排版知识，通过富文本编辑器即可完成论文撰写，一键生成符合规范的文档；
    \item 可扩展性：预留多学院模板接口、多语言支持接口、协同编辑接口扩展点。
\end{enumerate}
